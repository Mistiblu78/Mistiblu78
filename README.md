## Builds

**🗂️ Paralegal Services Capacity Queue** &nbsp;`private · coming soon`

A work management agent for a solo or small-team firm handling variable-complexity projects with sporadic client inflow and a fixed turnaround SLA.

The problem: you have 60 clients, a 3-day turnaround rule, and no way to see overload coming — because a simple agreed divorce and a contested decree with a business interest both enter the queue as "one project," but one takes 90 minutes and the other takes 6 hours.

What it does:
- **Intake classifier** — reads an incoming project request and suggests complexity weight (light / medium / heavy). Staff confirms or adjusts.
- **Capacity-aware queue** — shows workload by day so you can see Tuesday is already heavy before committing to a Tuesday deadline.
- **Morning digest** — plain daily summary of the next 72 hours: what's due, what's at risk, what needs to move.

`Google ADK` `Python` `Node.js` &nbsp;·&nbsp; In active development

<br/>

**🏛️ Firm OS — Texas Family Law Edition** &nbsp;`publishing soon`

A comprehensive offline operating system for DFW-based family law litigation, built in Google AI Studio.

- Vetted prompt repository — closed-loop AI prompts categorized by Texas family practice area
- Drafting clause library — certified family law boilerplate, indexed and ready to copy
- Firm knowledge wiki — SOPs, case workflows, state statutes, local court guidelines
- Compliance shield — Texas State Bar Opinion 705 audit logs built into standard copy triggers

🔒 Data never leaves the machine. No cloud syncing, no analytics, no external servers.
